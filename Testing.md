## Bugs

### Fixed Bugs

| Bug | Fix |
| --- | --- |
| Static files not loading on Heroku after deployment | `cloudinary_storage` in `INSTALLED_APPS` was intercepting and breaking Django's `collectstatic` command, resulting in 0 files copied. Removing `cloudinary_storage` from `INSTALLED_APPS` while keeping the `cloudinary` package resolved the issue. The `staticfiles/` folder was added to version control to ensure persistence across deployments. |
| `os.environ.setdefault` split across two lines in `manage.py` | The arguments were on separate lines which meant the function was never actually called. Django could not find the settings module causing `No Django settings specified` error. Fixed by moving both arguments onto the same line. |
| Cloudinary `Invalid Signature` error on image upload | A hidden character was present in the API secret when copied from the Cloudinary dashboard. Identified using `repr()` in the Django shell. Copying the secret fresh resolved the issue. |
| `instance=review` in `review_create` view causing `UnboundLocalError` | `instance` is only needed when editing an existing object. Removed from the create view. |
| Product images not loading on Heroku | Cloudinary was configured but images uploaded through admin were being stored on Heroku's ephemeral filesystem. Switched to using `CloudinaryField` directly in the Product model which routes images through Cloudinary correctly. |
| Page shifting horizontally due to scrollbar | On Windows the scrollbar takes up space and shifts content. Fixed by adding `scrollbar-gutter: stable` to the HTML element. |
| Bag showing €4.99 delivery when empty | Delivery cost was calculated even when the bag total was €0. Added a check for `total == 0` to set delivery to €0 when the bag is empty. |
| Forum login link causing 500 error for guest users | Template was using `{% url 'login' %}` instead of `{% url 'account_login' %}`. Fixed the URL reference. |
| `order_form` being overwritten by empty form after prefill | In the checkout view, an empty `OrderForm()` was being instantiated after the prefilled form, overwriting it. Removed the redundant instantiation. |

### Unfixed Bugs

> [!NOTE]
> There are no remaining unfixed bugs that I am aware of. The Lighthouse performance warning about unused CSS from Bootstrap and Font Awesome CDN files is outside of my control and has been noted as a future improvement in the README.
